# Cambios — sesión 2026-06-19

Resumen de todo lo implementado/ajustado en esta sesión. Detalle de uso de plantillas en
[`PLANTILLAS.md`](PLANTILLAS.md). Estado por feature en [`PLAN-MAESTRO.md`](PLAN-MAESTRO.md).

## Nuevas funciones

### Plantillas de código (R-D2) — `core/templates.lua`
- Store en disco editable: `~/.config/sap-nvim/templates/` (un `*.abap` por plantilla).
- Picker con preview (Telescope; fallback `vim.ui.select`).
- Guardar buffer o selección visual como plantilla desde la UI.
- **Parametrización estilo Eclipse:** al guardar, el nombre del objeto → `$OBJECT` (auto) y
  otros identificadores (grupo de funciones, tabla…) → tab-stops numerados y **espejados**
  (`${i:nombre}`/`$i`). Sirve para reutilizar un include entero (§6 de PLANTILLAS.md).
- Comandos: `:SapTemplate`, `:SapTemplateSave`, `:SapTemplatesDir`, `:SapTemplateEdit`.
- Atajos: grupo **`<leader>aP`** → `i` insertar · `s` guardar · `d` carpeta · `e` editar.
- Plantillas de ejemplo: `cabecera`, `cabecera_metodo`, `clase_test`, `select_loop`, `alv_sencillo`.

### Variables dinámicas con paridad Eclipse/ADT — `core/template_vars.lua`
- `$OBJECT $PACKAGE $SHORTTEXT $METHOD $AUTHOR $USER $SYSTEM $DATE $YEAR $MONTH $DAY $TIME $DOLLAR`.
- `$PACKAGE`/`$SHORTTEXT` reales: `template_vars.prime()` lee metadatos ADT async al abrir
  (vía `source.open`), sin tocar el hot-path del completado.
- `$METHOD` = método que contiene el cursor (`${enclosing_method}`).
- No tocan los tab-stops `${n}`. Probado offline.

### Completado contextual de keywords (R-A1) — `integrations/abap_local.lua`
- Detecta el contexto (firma de método / sección de clase / cuerpo / global) y prioriza
  (`score_offset`) los keywords propios: `IMPORTING`/`EXPORTING`/`RETURNING VALUE()`/
  `CHANGING` en una firma; `METHODS`/`CLASS-METHODS`/`DATA`/secciones en la definición.
- Compatible con la config de blink existente (no requiere cambios).

### Formateo nativo offline (R-B3) — `core/formatter.lua`
- Espaciado interior de paréntesis de **constructor** `#( ... )` (idempotente, solo `#(`;
  no toca offset/length `lv+0(10)` ni inline `DATA(x)`).
- Comandos `:SapFormat` (Pretty Printer de ADT si hay conexión) y `:SapFormatNative` (offline).
- (El Pretty Printer real de ADT, `format_via_adt`, ya existía; es el camino preferente online.)

## Arreglos

- **Swap file E325** en objetos cacheados: `source.open` abre con `:noswapfile` y
  `swapfile=false`. La caché es artefacto del plugin (la fuente real es SAP), no necesita swap.
